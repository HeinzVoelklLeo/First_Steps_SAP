# Project Instructions

## Leo HTML Default (Strict)

In this repository, for any request that creates or updates HTML output, use Leo branding by default.

Priority rules:
- Prefer the skill: leo-landing-html.
- Treat Leo branding as mandatory for HTML work unless the user explicitly requests a different style.
- If the user gives explicit non-Leo design instructions, follow the user request for that task only.

Override keyword:
- If the user includes the exact keyword `no-leo-style`, disable the Leo HTML default for that single request.
- The override applies only to the current task and does not change repository defaults.

Required Leo implementation for HTML tasks:
- Use project design sources:
  - Style_leogistics.html
  - 220201_Leo_Guidelines_V4.pptx
- Use Leo CI colors consistently:
  - #39769F (blue)
  - #172436 (deep blue)
  - #EFD100 (yellow)
  - #727272 (grey)
- Use Leo logo assets from this repository when available.
- Prefer Raleway for headings and Roboto for body text.
- Keep HTML responsive, presentation-ready, and clearly branded.

Scope guardrails:
- This strict default applies to HTML generation/editing tasks in this repository.
- Do not force Leo branding on non-HTML tasks unless requested.
